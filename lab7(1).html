Lab 7: PHP & MySQL CRUD Application

Objective: To develop a dynamic web application using PHP and MySQL that performs Create, Read, Update, and Delete (CRUD) operations on a database.

Section 1: Project Setup & Database

Project Structure

This lab uses multiple PHP files — one for each operation. All files are inside a lab7/ folder that you should place in your web server's document root (e.g., htdocs for XAMPP or www for WAMP).

lab7/
├── db.php         ← Database connection (shared by all files)
├── index.php      ← READ — Display all students in a table
├── create.php     ← CREATE — Form to add a new student
├── edit.php       ← UPDATE — Form to edit an existing student
└── delete.php     ← DELETE — Delete a student by ID
Prerequisites: You need a local server environment such as XAMPP or WAMP with Apache, PHP, and MySQL running. Open http://localhost/phpmyadmin to verify MySQL is working.

Example: How to Connect to MySQL & Create the Database

Before writing any CRUD code, you need a database and a table. Run the SQL below in phpMyAdmin (click the "SQL" tab) to create the database and table used throughout this lab.

-- Run this in phpMyAdmin → SQL tab

CREATE DATABASE IF NOT EXISTS lab7_studentdb;

USE lab7_studentdb;

CREATE TABLE IF NOT EXISTS students (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    email       VARCHAR(100) NOT NULL,
    course      VARCHAR(100) NOT NULL,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Next, the db.php file creates a reusable connection using MySQLi. This file is already written for you — read through the comments below:

<?php
// db.php — Database connection file
// Every other PHP file will include this with: require_once 'db.php';

$host   = "localhost";      // MySQL server address (localhost for XAMPP/WAMP)
$user   = "root";           // Default username for XAMPP/WAMP
$pass   = "";               // Default password is empty for XAMPP
$dbname = "lab7_studentdb"; // The database we created above

// Create a new MySQLi connection
$conn = new mysqli($host, $user, $pass, $dbname);

// Check if the connection failed
if ($conn->connect_error) {
    die("Connection failed: " . $conn->connect_error);
}
?>
Why require_once? It includes a file exactly once — if it's already been included, PHP skips it. This prevents accidentally creating multiple database connections. Every PHP file in this project starts with require_once 'db.php'; to get access to the $conn variable.
Security note: In a real application, you should never use root with no password. You should also use prepared statements (shown in this lab) instead of directly inserting user input into SQL queries, to prevent SQL injection attacks.

Key PHP & MySQL Concepts

Before starting the tasks, here's a quick reference of the PHP patterns you'll use:

Pattern Example
Prepared Statement (INSERT) $stmt = $conn->prepare("INSERT INTO t (col) VALUES (?)");
$stmt->bind_param("s", $value);
$stmt->execute();
SELECT all rows $result = $conn->query("SELECT * FROM t");
while ($row = $result->fetch_assoc()) { ... }
Prepared Statement (SELECT one) $stmt = $conn->prepare("SELECT * FROM t WHERE id = ?");
$stmt->bind_param("i", $id);
$stmt->execute();
$row = $stmt->get_result()->fetch_assoc();
Redirect after action header("Location: index.php");
exit;
Read form data $name = $_POST['name'];
Read URL parameter $id = $_GET['id'];
bind_param type codes "s" = string, "i" = integer, "d" = double, "b" = blob
"sss" = three strings, "isss" = int then three strings
Prepared statements are the safe way to run SQL with user input. The ? placeholder is filled by bind_param(), which automatically escapes the value — preventing SQL injection.

Section 2: CRUD Operations

Task 1: READ — Display All Students (index.php)

The main page queries the students table and displays every row in an HTML table. Each row has "Edit" and "Delete" links that pass the student's id via the URL (e.g., edit.php?id=3).

How it works:

  1. Include db.php to get the $conn connection
  2. Run SELECT * FROM students using $conn->query()
  3. Loop through results with while ($row = $result->fetch_assoc())
  4. Output each row inside <tr> tags using <?= $row['name'] ?>
  5. Add an "Add New Student" link to create.php and Edit/Delete links per row
Open lab7/index.php. Complete the PHP code to query all students and display them in the HTML table. Follow the step-by-step comments in the file.

Task 2: CREATE — Add a New Student (create.php)

This page has two jobs: show the form (on GET request), and process the form data (on POST request). When the form is submitted, PHP inserts a new row using a prepared statement, then redirects back to index.php.

How it works:

  1. Include db.php
  2. Check if the form was submitted: if ($_SERVER['REQUEST_METHOD'] === 'POST')
  3. Read form values from $_POST
  4. Prepare an INSERT statement with ? placeholders
  5. Bind parameters with bind_param("sss", ...) (3 strings: name, email, course)
  6. Execute and redirect to index.php
Why check $_SERVER['REQUEST_METHOD']? When a user first visits create.php, it's a GET request — just show the empty form. When they click "Submit", it's a POST request — that's when you process the data. One file handles both scenarios.
Open lab7/create.php. Complete the PHP block that handles the POST request — prepare the INSERT statement, bind the three parameters, execute, and redirect. Follow the step-by-step comments in the file.

Task 3: UPDATE — Edit an Existing Student (edit.php)

This page is more complex because it does three things: fetch the existing data (to pre-fill the form), display the form, and process the updated data on submission.

How it works:

  1. Include db.php
  2. Read the student ID from the URL: $id = $_GET['id'];
  3. On GET: Fetch the student's current data with a prepared SELECT and fill the form with their existing name, email, and course
  4. On POST: Read the updated values from $_POST, run a prepared UPDATE statement, and redirect to index.php
Pre-filling the form: Use the value attribute on inputs — e.g., <input value="<?= $student['name'] ?>">. For textareas, put the value between the opening and closing tags.
Open lab7/edit.php. Complete both parts: the GET handler (fetch and pre-fill) and the POST handler (update and redirect). Follow the step-by-step comments in the file.

Task 4: DELETE — Remove a Student (delete.php)

The simplest operation — receive an id via the URL, delete that row from the database, and redirect back to the list.

How it works:

  1. Include db.php
  2. Read the student ID from the URL: $id = $_GET['id'];
  3. Prepare a DELETE statement with a ? placeholder
  4. Bind the ID parameter with bind_param("i", $id)
  5. Execute and redirect to index.php
In a real application, you should never delete records via a GET request (links). GET requests should only read data. A proper implementation would use a POST form with a confirmation step. We use GET here for simplicity.
Open lab7/delete.php. Complete the PHP code to delete the student and redirect. Follow the step-by-step comments in the file.

Summary of Concepts

Concept What You Learned
MySQLi Connection Create a database connection with new mysqli() and share it across files using require_once.
Prepared Statements Use prepare(), bind_param(), and execute() to safely run SQL with user input.
$_POST and $_GET $_POST reads data from form submissions. $_GET reads data from URL parameters.
CRUD Operations The four fundamental database operations: Create (INSERT), Read (SELECT), Update (UPDATE), Delete (DELETE).
Request Method Check $_SERVER['REQUEST_METHOD'] tells you if the request is GET or POST — use it to handle forms.
Redirect with header() header("Location: index.php"); exit; sends the user to another page after a database action.
fetch_assoc() Returns a row as an associative array: $row['name'], $row['email'], etc.
PHP in HTML Use <?= $var ?> (short echo) to output PHP values directly inside HTML markup.
