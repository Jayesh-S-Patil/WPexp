Task 1: Select the Form Element

Before we can work with any HTML element in JavaScript, we need to select it from the DOM (Document Object Model).

Use document.getElementById() to select the form and store it in a constant. 
// ============================================================
// TASK 1: DOM SELECTION
// ============================================================
// document.getElementById('id') finds an HTML element by its id attribute
// We use 'const' because this reference won't change

const form = document.getElementById('registrationForm');

// Verify it worked by logging to console (open browser DevTools with F12)
console.log('Form element:', form);
DOM Selection: The DOM is a tree representation of your HTML. JavaScript uses methods like getElementById(), querySelector(), and getElementsByClassName() to find and access elements.

Task 2: Add Form Submit Event Listener

We need to listen for when the user submits the form, then run our validation code.

Add an event listener that listens for the 'submit' event and prevents the default form submission. 
// ============================================================
// TASK 2: EVENT HANDLING
// ============================================================
// addEventListener(eventType, handlerFunction) attaches code to run when an event occurs
// Common events: 'click', 'submit', 'input', 'change', 'keydown'
//
// event.preventDefault() stops the browser's default action (form submission/page reload)
// This lets us validate BEFORE actually submitting

    event.preventDefault();  // Stop the form from submitting normally

});
Event Handling: Events are signals that something happened (click, keypress, form submit). We use addEventListener() to run code when events occur. The event object contains info about what happened.

Task 3: Get Input Values

Inside our event handler, we need to get the values the user typed into each field.

Select each input field and get its value using the .value property. 
// ============================================================
// TASK 3: GETTING INPUT VALUES
// ============================================================
// Inside your submit event handler, add:

form.addEventListener('submit', function(event) {
    event.preventDefault();

    // Get input values using .value property
    // .trim() removes whitespace from start and end of string
    const username = document.getElementById('username').value.trim();
    const email = document.getElementById('email').value.trim();
    const password = document.getElementById('password').value;
    const confirmPassword = document.getElementById('confirmPassword').value;

    // Log values to verify (check browser console)
    console.log('Username:', username);
    console.log('Email:', email);
    console.log('Password length:', password.length);
});
Accessing Values: Form inputs have a .value property containing what the user typed. Using .trim() removes accidental spaces. We don't trim passwords since spaces might be intentional.

Task 4: Create Validation Helper Functions

Let's create reusable arrow functions (ES6 feature) to check if values are valid.

Create arrow functions to validate: empty fields, email format, password length, and matching passwords. 
// ============================================================
// TASK 4: ARROW FUNCTIONS (ES6)
// ============================================================
// Arrow functions: const name = (params) => expression;
// Shorter syntax than traditional functions
// Single expression automatically returns (no 'return' keyword needed)

// Check if a value is empty
const isEmpty = (value) => value === '';

// Check if email format is valid using a Regular Expression
// Regular expressions (regex) are patterns used to match text
// This pattern checks for: something@something.something
const isValidEmail = (email) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);

// Check if password is at least 8 characters
const isValidPassword = (password) => password.length >= 8;

// Check if two passwords match
const doPasswordsMatch = (pass1, pass2) => pass1 === pass2;

// Test them in console:
console.log('isEmpty test:', isEmpty(''));           // true
console.log('isEmpty test:', isEmpty('hello'));      // false
console.log('Email test:', isValidEmail('a@b.com')); // true
console.log('Email test:', isValidEmail('invalid')); // false
Arrow Functions (ES6): A shorter way to write functions. const add = (a, b) => a + b; is equivalent to function add(a, b) { return a + b; }. Great for small, single-purpose functions.

Task 5: Create Functions to Show/Clear Errors

We need functions to display error messages next to invalid fields and clear them when fixed.

Create functions to show and clear error messages by manipulating the DOM. 
// ============================================================
// TASK 5: DOM MANIPULATION - Showing/Clearing Errors
// ============================================================
// .textContent sets the text inside an element
// .style.property lets us change CSS properties directly

// Show an error message for a specific field
const showError = (fieldId, message) => {
    // Find the error div (we named them fieldId + '-error')
    const errorDiv = document.getElementById(fieldId + '-error');
    errorDiv.textContent = message;

    // Also highlight the input field with red border
    const inputField = document.getElementById(fieldId);
    inputField.style.borderColor = '#e74c3c';
};

// Clear the error message for a specific field
const clearError = (fieldId) => {
    const errorDiv = document.getElementById(fieldId + '-error');
    errorDiv.textContent = '';

    // Reset border to default
    const inputField = document.getElementById(fieldId);
    inputField.style.borderColor = '#bdc3c7';
};

// Clear all error messages
const clearAllErrors = () => {
    // Array of all field IDs - we loop through each one
    ['username', 'email', 'password', 'confirmPassword'].forEach(fieldId => {
        clearError(fieldId);
    });
    document.getElementById('success-message').textContent = '';
};
DOM Manipulation: We can change what's displayed on the page by modifying element properties. .textContent changes text, .style changes CSS, .classList adds/removes classes.
